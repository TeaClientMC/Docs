# Commiting
When you commit you must Use the [Covential Commits Format](https://www.conventionalcommits.org/en/v1.0.0-beta.2/)
```
Type: [Optional: Scope]: Description

Body/Description

Optional: Description
```
##### Types:
> Fixes: For fixing code that is not working properly.
 
> Bugs: For bugs that are making the application fail/glitch.

> Breaking Changes/Major Changes: Added a major Changes

> Patches: Like a patch to patch bugs + fixes.


**Note: That Some commits require you to use the [Semantic Versioning.](https://www.geeksforgeeks.org/introduction-semantic-versioning/)**
