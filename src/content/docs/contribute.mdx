#### Please adhere to our [Code-of-Conduct](https://teaclient.net/policies/code-of-conduct)

##### Not adhering to the rules will lead to your Pull Request (PR) being closed.

# Repository Forking Steps

**Step 1:** Click the fork button at the top of this page to create a fork of this repository.
**Step 2:** This action creates a duplicate of the repository in your account.

Provide a name and description for the forked repository.
Optionally, if you don't want to just copy the `main` branch, you can untick "Copy the `main` branch only."

Finally, click "Create Fork."

# Commits

Commit messages should follow the [Conventional Commits Format](https://www.conventionalcommits.org/en/v1.0.0-beta.2/):
```
Type: [Optional: Scope]: Description

Body/Description

Optional: Additional Details
```

##### Commit types include:

> **Fixes:** Applying a patch to non-functional code.

> **Bugs:** Addressing bugs causing the app to crash or behave unexpectedly.

> **Breaking Changes/Major Changes:** Implementing significant changes to the code.

> **Patches:** Applying patches to address bugs and/or fixes.

**Note: Some commits might necessitate the use of [Semantic Versioning](https://www.geeksforgeeks.org/introduction-semantic-versioning/).**
